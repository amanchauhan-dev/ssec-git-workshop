# Git & GitHub Workshop

## Beginner → Advanced

# What is Version Control?

Version Control is a system that tracks changes in files over time.

It allows:

- Saving versions of files
- Tracking changes
- Restoring old versions
- Collaborating with teams

---

# What is Git?

Git is a **distributed version control system**.

Created in **2005 by Linus Torvalds**.

Git helps:

- Track changes
- Save history
- Work with teams
- Manage projects safely

---

# Why Version Control Matters

Without version control:

- Files get overwritten
- No history tracking
- Hard collaboration
- Lost work
- Manual backups required

Version control solves these issues.

---

# The Problem Before Git

Typical folder chaos:

```
project/
project_v1/
project_v2/
project_final/
project_final_latest/
project_final_USE_THIS/
```

Problems:

- Confusion
- Lost work
- Hard to manage

---

# How Git Solves These Problems

Git provides:

- Version tracking
- Safe backups
- Collaboration tools
- History management
- Easy rollback

Instead of folders:

```
project/
.git/
```

Git stores history internally.

---

# What is GitHub?

GitHub is a **cloud platform**.

Used to:

- Store repositories
- Share projects
- Collaborate
- Review code
- Manage teams

GitHub works together with Git.

---

# Git vs GitHub

| Feature  | Git            | GitHub         |
| -------- | -------------- | -------------- |
| Type     | Local Tool     | Cloud Platform |
| Internet | No             | Yes            |
| Purpose  | Track versions | Collaboration  |
| Runs On  | Computer       | Web            |

Git → Local
GitHub → Online

# Installing Git — Windows

Steps:

1. Go to:
   [https://git-scm.com/download/win](https://git-scm.com/download/win)

2. Download installer

3. Run setup

4. Accept default settings

Verify:

```bash
git --version
```

Verify:



Verify:


# First-Time Configuration

Set identity:

```bash
git config --global user.name "Your Name"

git config --global user.email "you@email.com"
```

Set default branch:

```bash
git config --global init.defaultBranch main
```

Check settings:

```bash
git config --list
```

# Git Three-Area Architecture

Git has three areas:

1. Working Directory
2. Staging Area
3. Repository

Workflow:

```
edit → add → commit
```

# Create Your First Repository

Commands:

```bash
mkdir project
cd project

git init
```

Check status:

```bash
git status
```

Repository created.

# Essential Git Commands

Core commands:

```bash
git status
git add
git commit
git log
git diff
```

Used daily in projects.

---

# git status

Shows:

- Modified files
- Untracked files
- Staged files

Example:

```bash
git status
```

Most commonly used command.

---

# git add

Stages files for commit.

Example:

```bash
git add file.txt
```

Add all files:

```bash
git add .
```

Moves files to staging area.

---

# git commit

Creates a snapshot of changes.

Example:

```bash
git commit -m "Add homepage"
```

Each commit saves a version.

---

# git log

Shows commit history.

Example:

```bash
git log
```

Short version:

```bash
git log --oneline
```

Used to view history.

---

# .gitignore File

Prevents unwanted files from being tracked.

Common examples:

```
node_modules/
.env
dist/
build/
*.log
__pycache__/
```

Important for security.
